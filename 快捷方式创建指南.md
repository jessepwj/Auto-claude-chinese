## 🖱️ 创建桌面快捷方式（推荐）

### 方法一：手动创建（最简单）

**步骤：**
1. 找到 `快速启动.bat` 文件
2. **右键** → **发送到** → **桌面快捷方式**
3. 完成！现在桌面上有快捷方式了

**重命名快捷方式：**
- 右键快捷方式 → **重命名** → 改成 `Auto Claude 中文版`

### 方法二：自定义图标（更专业）

**步骤：**
1. 右键桌面 → **新建** → **快捷方式**
2. 在"请键入对象的位置"中输入：
   ```
   "E:\aigc内容整理\Auto-Claude-develop\快速启动.bat"
   ```
   （⚠️ 注意：引号要保留，路径替换成您的实际路径）
3. 点击 **下一步**
4. 输入名称：`Auto Claude 中文版`
5. 点击 **完成**

**设置自定义图标（可选）：**
1. 右键快捷方式 → **属性**
2. 点击 **更改图标**
3. 浏览到项目目录中的图标文件（如果有）或选择系统图标
4. 点击 **确定** → **应用** → **确定**

### 方法三：使用PowerShell脚本

**如果您喜欢PowerShell：**
1. 找到 `启动.ps1`
2. 右键 → **发送到** → **桌面快捷方式**
3. 右键快捷方式 → **属性**
4. 在"目标"中添加PowerShell：
   ```
   powershell.exe -ExecutionPolicy Bypass -File "E:\aigc内容整理\Auto-Claude-develop\启动.ps1"
   ```

## 🎨 自定义启动选项

### 修改启动模式

编辑 `快速启动.bat` 文件：

**默认（开发模式）：**
```batch
npm run dev
```

**改为生产模式：**
```batch
npm start
```

**改为调试模式：**
```batch
npm run dev:debug
```

### 添加环境变量

在启动脚本中添加：
```batch
set DEBUG=true
set NODE_ENV=development
npm run dev
```

## 🔧 首次运行准备

### 1. 安装依赖（只需一次）

**方法A：自动安装**
- 运行任一启动脚本，会自动检测并安装依赖

**方法B：手动安装**
```bash
npm run install:all
```

### 2. 配置API密钥

**编辑配置文件：**
```
apps\backend\.env
```

```env

# 或使用 Anthropic API Key

```

**获取Token：**
```bash
# 使用Claude Code CLI

```

## ⚡ 常见问题

### Q1: 双击启动脚本后窗口一闪而过

**原因：** 缺少依赖或配置错误

**解决：**
1. 右键脚本 → **编辑**
2. 检查是否有错误提示
3. 或使用 `启动-开发模式.bat`（有详细错误信息）

### Q2: 提示"找不到npm命令"

**原因：** Node.js未安装或未添加到PATH

**解决：**
1. 安装 Node.js 24+：https://nodejs.org/
2. 重启电脑
3. 重新运行启动脚本

### Q3: 端口被占用

**错误信息：** `Port 5173 is already in use`

**解决：**
```bash
# 方法1：关闭占用端口的程序
netstat -ano | findstr :5173
taskkill /PID <进程ID> /F

# 方法2：修改端口（编辑 vite.config.ts）
```

### Q4: PowerShell脚本无法运行

**错误：** `无法加载文件，因为在此系统上禁止运行脚本`

**解决：**
```powershell
# 以管理员身份运行PowerShell
Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
```

### Q5: 启动后应用窗口不出现

**可能原因：**
1. 后端进程未启动 - 检查 `.env` 配置
2. 前端构建失败 - 查看终端错误信息
3. 防火墙阻止 - 允许Node.js通过防火墙
